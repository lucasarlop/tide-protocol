# Quality

- Prefer the smallest change that solves the real problem.
- Do not expand scope.
- Do not add abstractions for hypothetical flexibility.
- No new dependency without explicit approval.
- Preserve unrelated user changes.
- Targeted validation first. Full suite only when justified.
- Failed, skipped, timed-out, or inconclusive validation is not success.
- Tide evidence is temporary. Do not version it.
